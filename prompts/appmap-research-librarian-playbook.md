# AppMap Research Librarian Playbook

## Overview

Use the AppMap Research Librarian as a gated staging workflow between open-ended research agents and deterministic AppMap ingest.

The librarian wrapper itself is offline: it creates campaign files, briefs a research scout, briefs a validator, validates local seed data, and plans the AppMap ingest command. It does **not** search the web, spider Google, crawl targets, launch browsers, or probe application infrastructure.

## Intended Agent Chain

1. **Research scout agent**
   - Receives `scout_brief.md`.
   - Searches externally using its own allowed tools/context.
   - Produces curated source candidates in `sources.todo.jsonl`.
   - Explains why each source matters for the requested category/class.

2. **Validator agent**
   - Receives `validator_brief.md` and `sources.todo.jsonl`.
   - Rejects weak, duplicate, SEO, uncited, or irrelevant sources.
   - Writes `validated_research_seed.json` with structured `sources` and `technique_packs`.

3. **Librarian validation**
   - Runs local seed validation with no network access.
   - Produces `validation_report.json` with counts, categories, and errors.

4. **AppMap ingest**
   - Prefer local deterministic ingest:
     `--research-mode local --research-seed validated_research_seed.json`
   - Optionally plan explicit URL ingest:
     `--research-mode web --research-source-url <validated-url>`

## Commands

Create a campaign:

```bash
python3 agents/appmap_research_librarian.py init <program> \
  --category electron-ipc \
  --research-query electron rce \
  --target-kind electron-exe
```

Validate the validator-produced seed:

```bash
python3 agents/appmap_research_librarian.py validate <campaign_dir>
```

Plan deterministic local AppMap ingest:

```bash
python3 agents/appmap_research_librarian.py plan-appmap <campaign_dir> <target_path> \
  --write-specs \
  --output-mode canonical \
  --family binaries \
  --lane exe
```

Plan explicit validated-source web ingest:

```bash
python3 agents/appmap_research_librarian.py plan-appmap <campaign_dir> <target_path> \
  --use-web-sources
```

## Campaign Artifacts

Each campaign contains:

- `manifest.json` — campaign metadata, query/category normalization, network policy, and paths.
- `scout_brief.md` — prompt for the source-finding research agent.
- `sources.todo.jsonl` — scout output, one candidate source per JSONL row.
- `validator_brief.md` — prompt for the validation/structuring agent.
- `validated_research_seed.json` — structured AppMap local seed data.
- `validation_report.json` — generated by `validate`.
- `plan_appmap_command.txt` — generated by `plan-appmap`.
- `README.md` — campaign-local instructions.

## Seed Contract

Use this structure for `validated_research_seed.json`:

```json
{
  "sources": [
    {
      "id": "S0001",
      "title": "Source title",
      "url": "https://example.org/source",
      "summary": "What this source proves or supports."
    }
  ],
  "technique_packs": [
    {
      "id": "electron-ipc-rce",
      "title": "Electron IPC RCE pattern",
      "summary": "Concrete technique and preconditions.",
      "vulnerability_pack": "rce",
      "target_pack_keys": ["electron"],
      "applicable_surface_kinds": ["ipc"],
      "source_ids": ["S0001"],
      "guidance": ["What AppMap should look for in local code."]
    }
  ]
}
```

Validation expects:

- every source has `id`, `title`, and `url` or `local_path`
- every technique has `title`, `summary`, `vulnerability_pack`, and `source_ids`
- every technique cites known sources
- every technique declares `target_pack_keys` and `applicable_surface_kinds`, unless `applies_to_all` is deliberately set
- `plan-appmap` requires at least one validated source and one validated technique pack

## Safety and Reproducibility Rules

- Do not put web search or crawling inside AppMap.
- Do not let unvalidated research flow directly into generated AppMap hypotheses.
- Keep the source-finding and validation agent roles separate when possible.
- Prefer local seed ingest for replayability.
- Use web-source ingest only for explicit, validator-approved HTTPS URLs.
- Keep all generated files in the campaign directory so another agent can replay or audit the decision chain.
